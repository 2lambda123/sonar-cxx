Public APIs have to be documented in order to be used by customers.
APIs documentation is typically generated using a tool, such as Doxygen.

Exceptions that do not require documentation:

  • Member functions marked with identifier override. Assumption is that they are documented in the base class.
  • Member functions marked with specifier delete or default.
  • Function definitions with a qualified name outside of a class. Assumption is that they are already documented inside of the class.

Following items are considered as public API:

  • classes/structures/templates
  • class members (public and protected)
  • structure members
  • union members
  • enumerations
  • enumeration values
  • typedefs
  • alias declaration (using MyAlias = int;)
  • functions
  • variables

The following code illustrates a well documented class:

/**
  class documentation
 */
class DocumentedClass {
  public:

    DocumentedClass() = default;
    DocumentedClass& operator(const DocumentedClass&) = delete;

    int commentedVar; ///< documentation

    void apiFromBaseClass() = override;

    /**
      apiMethod documentation
     */
    void apiMethod();

  private:
    void notInApiMethod();
};

void DocumentedClass::apiMethod()
{
   // documentation should be inside of the class
}
